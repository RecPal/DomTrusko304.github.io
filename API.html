Message Types

We have various message types that need to be transmitted through this standard. The nature of the header/footer system allows for dynamic message sizes. The types of messages and lengths are listed below.
All messages are also recieved by the wifi system for live-display to the operator via the web-interface. All messages can also be spoofed by the web-interface to overwrite command inputs by the user
Each subsystem address is laid out below:
Member Subsystem Address
Dominick Trusko Pitch-Control and Wifi 0x06
Cory Lewis Fan Speed Controller 0x07
Adam Quan Human Interface 0x08
Emerson Wall Windspeed Display 0x09
All All Subsystems 0x10
For the "Example" section, values listed in BLUE are common for all messages (Headers/Footers)
TX/RX Message Name Data type Range Size Description Example
TX Ready to Recieve uint_8 [255] 8 Bits [56 Bits Total] Notifies subsystems that ready commands can be recieved [0x41][0x5a] [0x06][0x07][0xFF] [0x59][0x42]
TX Initialize Clock uint_8 [255] 8 Bits [56 Bits Total] Tells all devices to start their send clock [0x41][0x5a] [0x06][0x10][0xFF] [0x59][0x42]
TX Emergency Stop uint_8 [244] 8 Bits [56 Bits Total] Commands all devices to cease operation [0x41][0x5a] [0x06][0x10][0xF4] [0x59][0x42]
RX No Report uint_8 [0] 8 Bits [56 Bits Total] Indicates to master device that no new information is available [0x41][0x5a] [0x08][0x6][0x00] [0x59][0x42]
RX Airspeed uint_8 [0-255] 8 Bits [56 Bits Total] Airspeed reporting from Windspeed subsystem [0x41][0x5a] [0x09][0x6][0x43] [0x59][0x42]
Reset System uint_8 0xAE 8 Bits Wifi System All Devices
Emergency Stop uint_8 0x99 8 Bits Wifi System All Devices
